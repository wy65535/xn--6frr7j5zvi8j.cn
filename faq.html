如何充值？

🔧 API使用

如何获取API密钥？

登录硅基流动控制台 → 进入"API密钥管理" → 点击"创建新密钥" → 复制保存密钥。请妥善保管API密钥，不要泄露给他人。

硅基流动API兼容OpenAI吗？

完全兼容！硅基流动使用OpenAI标准协议，您可以直接使用OpenAI的SDK和工具。只需修改两处配置：

  • API密钥：替换为硅基流动的密钥
  • API地址：设置为 https://api.siliconflow.cn/v1

其他代码完全不需要修改。查看迁移教程。

API有请求频率限制吗？

有合理的频率限制，但对大多数用户来说完全够用：

  • 免费用户：60次/分钟
  • 付费用户：根据充值金额自动提升
  • 企业用户：可定制专属限额

如需更高频率，请联系客服申请提额。

API调用失败怎么办？

常见错误及解决方法：

  • 401错误：API密钥无效，请检查密钥是否正确
  • 429错误：请求过于频繁，请降低频率或申请提额
  • timeout：网络超时，建议使用流式输出
  • 500错误：服务器错误，请稍后重试或联系支持

更多问题请查看使用教程或联系技术支持。

支持流式输出吗？

支持！流式输出（Stream）可以让用户实时看到AI回复过程，提供更好的用户体验。在API调用时设置 stream=True 即可启用。查看代码示例。

可以在浏览器中直接调用吗？

可以，但不推荐。在浏览器前端直接调用会暴露API密钥，存在安全风险。推荐做法是：

  • 前端调用自己的后端API
  • 后端使用硅基流动API
  • API密钥存储在后端环境变量中

支持哪些编程语言？

硅基流动提供标准HTTP API，支持所有编程语言。官方推荐：

  • Python：使用 openai 库
  • JavaScript/Node.js：使用 openai npm包
  • Java：使用HTTP客户端或第三方SDK
  • Go：使用HTTP客户端或第三方SDK
  • PHP/Ruby/C#等：使用对应的HTTP库

查看多语言示例。

如何实现多轮对话？

在messages数组中包含完整的对话历史即可：

messages = [ {"role": "system", "content": "你是AI助手"}, {"role": "user", "content": "你好"}, {"role": "assistant", "content": "你好！有什么可以帮你的？"}, {"role": "user", "content": "介绍一下硅基流动"} ]

详细教程请查看使用教程。

支持函数调用（Function Calling）吗？

支持！硅基流动完整支持OpenAI的Function Calling功能，可以让AI自动调用您定义的函数。适用于需要AI与外部系统交互的场景，如查询数据库、调用API等。

如何控制输出长度？

使用 max_tokens 参数限制输出长度。例如：max_tokens=1000 表示最多输出1000个tokens。合理设置可以节省费用。

什么是temperature参数？

temperature控制输出的随机性：

  • 0-0.3：输出更确定、一致，适合事实性任务
  • 0.7-0.8：平衡创造性和一致性，推荐值
  • 0.9-1.0：输出更有创造性和多样性，适合创意写作

如何处理超长文本？

不同模型支持的最大context长度不同：

  • GPT-3.5 Turbo：4K tokens
  • GPT-4 Turbo：128K tokens
  • Claude 3.5：200K tokens

如果文本超过限制，需要进行分段处理或使用支持更长context的模型。
